# Day 35 —— 数据配比实验：设计 3 组配比 + 生成数据集

> 目标：探索不同数据配比对微调效果的影响，设计并生成 3 组对照数据集。

> **硬件需求**：需要 >= 24GB 显存的 GPU。如果没有 GPU，可用 Colab Pro (A100) 或 Hugging Face Spaces 替代。

---

## Task 80：数据配比方案设计

**产出物**：`data/finetuning/data_ratio_experiment_design.md`

- [ ] 80.1 确定 3 组配比方案：A 组（通用 70% + 领域 30%）、B 组（通用 40% + 领域 60%）、C 组（纯领域 100%）（20min）
- [ ] 80.2 定义"通用数据"来源：Alpaca、OpenOrca 等公开指令微调数据集（15min）
- [ ] 80.3 定义"领域数据"来源：目标领域的专业 QA、文档转指令、合成数据生成（20min）
- [ ] 80.4 明确评估方式：统一用一组领域测试题评测 3 个微调后模型的效果（15min）

## Task 81：生成 3 组配比数据集

**产出物**：`data/finetuning/sft_ratio_a.jsonl`、`data/finetuning/sft_ratio_b.jsonl`、`data/finetuning/sft_ratio_c.jsonl`

- [ ] 81.1 编写数据配比脚本：按设定比例混合通用数据和领域数据，确保每组 500 条（30min）
- [ ] 81.2 对每组数据做格式验证和去重，确保训练质量一致（15min）
- [ ] 81.3 统计每组数据的 token 分布、领域比例、指令类型分布（20min）
- [ ] 81.4 保存 3 组数据集的统计报告到 `data/finetuning/` 目录下各自的 stats 文件（15min）
- [ ] 81.5 撰写实验计划：后续用这 3 组数据分别微调，对比领域任务表现差异（15min）
